测试用例1：Git命令多行代码（包含空行）

以下是用户提供的多行Git命令代码块，应该能够正确换行显示并保留空行：

# 首先初始化git仓库
git init

# 然后把所有文件改动添加到暂存区
git add .

# 然后提交代码到本地git，双引号的内容可以自己随便写，前面是固定写法。
git commit -m "0.0.1 第一个版本内容"

# 接着，将本地默认主分支，重命名为 main (这是一个好习惯)，固定写法，（只需要在初始化做一次）。
git branch -M main

# 关联远程仓库（只需要在初始化做一次）
git remote add origin https://github.com/zhangsan/online-store

# 将本地的 main 分支，推送远程仓库并建立连接关系
git push -u origin main

    }));

测试用例4：Python多行代码（包含空行和缩进）

def calculate_fibonacci(n):
    """计算斐波那契数列的第n项"""
    if n <= 1:
        return n
    
    # 使用迭代方法计算
    a, b = 0, 1
    for i in range(2, n + 1):
        a, b = b, a + b
    
    return b


# 测试函数
if __name__ == "__main__":
    
    # 打印前10个斐波那契数
    for i in range(10):
        result = calculate_fibonacci(i)
        print(f"fibonacci({i}) = {result}")
    
    print("\n计算完成！")

验证标准

如果修复成功，以上所有代码块应该：

  • 每行代码都在单独的行上显示
  • 空行被正确保留和显示
  • 代码的缩进格式保持不变
  • 注释和代码分行显示
  • 不会出现所有内容挤在一行的情况
